Test de Sécurité avec CoercedPotato

Lors de mon stage chez Cyklad, j’ai testé CoercedPotato, un exploit permettant une élévation de privilèges sous Windows via des failles RPC. L’objectif était d’évaluer la protection de CrowdStrike.

Résultats :

  • Création d’un serveur pipe → Réussi, détecté par CrowdStrike.
  • Coercition d’authentification → Réussi, alerte générée.
  • Exécution d’un processus SYSTEM → Bloqué par CrowdStrike.

Conclusion :

CoercedPotato exploite les failles RPC pour obtenir des droits SYSTEM, mais CrowdStrike détecte et bloque efficacement l’attaque.

Compétences validées :

  • Répondre aux incidents : test d'attaque détecté et bloqué par un EDR.
  • Gérer le patrimoine informatique : compréhension des vulnérabilités et des privilèges système.
  • Organiser son développement professionnel : veille sur une conférence (SSTIC) et usage d’un outil d’analyse (MITRE ATT&CK).

Projet : Analyse de logs FortiGate & génération de règles firewall

Ce projet consiste à exploiter un script Python pour analyser les logs réseau FortiGate issus d’une règle “ALL” afin de générer automatiquement des règles précises et optimisées pour le pare-feu.

Objectifs :

  • Filtrer les flux autorisés à partir des logs (exclusion des flux “deny”).
  • Calculer des statistiques pour détecter les flux pertinents.
  • Générer automatiquement des objets et des règles FortiGate.
  • Créer une règle dédiée aux flux HTTP/HTTPS vers IP publiques.

Résultats :

  • Script fonctionnel capable de traiter et classifier les flux réseau.
  • Export automatique de règles config firewall policy.
  • Gain de temps et réduction des erreurs dans la configuration.

Compétences validées :

  • Gérer le patrimoine informatique : exploitation de logs et configuration réseau automatisée.
  • Répondre aux incidents : détection et traitement des flux autorisés.
  • Travailler en mode projet : développement, tests et intégration dans un outil interne (Panda).
  • Organiser son développement professionnel : gestion de version et publication via GitLab.

Conclusion :

Ce projet m’a permis de combiner Python, analyse réseau et sécurité pour produire un outil automatisé utile à l’équipe. L'intégration dans Panda et le partage sur GitLab assurent sa réutilisabilité.
